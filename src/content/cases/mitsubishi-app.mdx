<Chapter n="01" title="O desafio">

O produto acumulava informação sem hierarquia: o usuário recebia dados, não direção. A tela inicial competia por atenção em vez de guiar a próxima ação.

O briefing pedia uma interface que parecesse simples sem ser rasa — precisão de leitura para um público que decide com base no que vê.

</Chapter>

<Figure variant="split" caption="Fig. 01 — Antes / depois da hierarquia de entrada." />

<Chapter n="02" title="A solução">

Redesenhamos em torno de um sistema de grid rígido: cada módulo com um trabalho, cada número com um lugar. Tipografia grotesca para os dados, mono para os rótulos.

O fluxo de onboarding foi reduzido a três decisões. O resto some até ser necessário.

</Chapter>

<Figure variant="radial" caption="Fig. 02 — Sistema de componentes e estados." />

<Chapter n="03" title="O resultado">

A clareza virou métrica. Menos toques até a ação, mais retorno ao app, e uma leitura que o time clínico validou de primeira.

</Chapter>
